（1）通信效率与信息密度对比实验（针对json格式的data schema）
通过验证以状态为中心的结构化通信相比自然语言通信在降低信息冗余和提高精细度方面的优势来测试“JSON Patch”。

- 对照组： 采用自然语言对话模式的多智能体系统
  - AutoGen
  - LangGraph
- 实验组： 本论文系统。
- 实验任务： 使用开源的数据集测试并分析关键指标
- 数据集：
  **找一个数据集选一个表现最好的子集出来都行，之后的实验也可以沿用这几个子集的内容**
  - ALFWorld（文本房间探索，状态就是房间里的物品和位置）
  - WebArena（网页导航，状态可以抽象为精简版的 DOM 树）
- 关键指标：
  - token消耗量：不同系统每次任务平均消耗token量
  - 语义漂移率： 衡量接收方对发送方意图理解的准确性
    - 统计接收方 Agent 因为误解发送方意图而导致产生“无效操作”或“错误 Patch”的次数
    - 通过错误率衡量： 
      $$
      \frac{\text{接收方误解导致的无效 Patch 次数}}{\text{总通信次数}}
      $$
    - 通过相关性衡量： 使用 LLM 作为评委，对发送方意图与接收方动作的一致性进行打分（0或1）。如果一致性评分高于 0.95，则证明语义传递精准

（2）确定性内核通过 Schema 强制校验对比实验【ok】

- 实验组： 本实验系统。
- 对照组： 关闭内核的校验逻辑。Worker Agent 提交的 JSON Patch 不经过任何格式或逻辑检查（取消kernel的审查功能），直接应用到全局状态树中 。
- 实验内容：
  - 注入攻击： 在 Worker Agent 的 Prompt 中加入诱导性指令，诱使它生成不符合规范的字段。然后查看能否输出符合预期的内容。数据集公开的数据集上面手动注入一部分
- 数据集：
  使用实验一的数据集：WebArena、ALFWorld
  - 手动注入策略：
    - 在Prompt中加入"忽略Schema约束"的指令
    - 注入格式错误的JSON示例
    - 注入类型不匹配的字段值
    - 注入比例：10-20%的测试样本
- 拦截率：内核成功拦截无效Patch的比例
  - 计算公式： 
    $$
    \text{拦截率} = \frac{\text{被拦截的无效Patch数}}{\text{总无效Patch数}} \times 100
    $$

（3） Context Slicing 的对比实验
目的：验证Context Slicing 可以缓解在长任务当中的幻觉问题

- 实验组： 本实验系统。
- 对照组： 每次给agent的信息都是完整的，而没有经过input_schema的过滤
- 实验内容：进行长文档跨段落分析
- 数据集：
  - WebArena：使用的网页文本庞大
  - ALFWorld
- 关键指标：
  - 准确率：正确回答问题的比例
  - 信息检索精度： 
    $$
    \text{Precision} =
    \frac{\text{提取的相关信息片段数}}{\text{提取的总信息片段数}} \times 100
    $$
